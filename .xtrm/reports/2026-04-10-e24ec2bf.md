## Summary

Completed epic `unitAI-t4ss` — coordinator CLI-native redesign — across 4 sequential waves. Wave 1 added 5 new `sp node` subcommands (spawn-member, create-bead, complete, wait-phase, status). ## Code Changes

### CLI commands (src/cli/node.ts)
Added 5 new `sp node` subcommands: `spawn-member`, `create-bead`, `complete`, `wait-phase`, `status`. Each returns `{ ok: true/false, ... }` JSON. Includes `--strategy` validation (pr|manual), JSON error output for parse failures via `emitNodeCommandError()`, and `hasJsonFlag()` helper.

### Supervisor cleanup (src/specialist/node-supervisor.ts)
Removed `handleCoordinatorOutput()` (JSON blob parsing), `buildCoordinatorRepairPrompt()` (3-attempt repair loop), and coordinator output Zod validation. Exported action functions (`spawnDynamicMember`, `executeCreateBeadAction`, `executeCompleteNodeAction`) as CLI backends. -875 lines net. Kept action queue, dispatch, events, watchdog.

### Contract renderers (src/specialist/node-contract.ts)
Rewrote `renderForSystemPrompt()`, `renderForFirstTurnContext()`, `renderForResumePayload()`, `renderForDocs()` to describe CLI commands instead of JSON schema. Removed `coordinatorOutputSchema` and `buildCoordinatorOutputJsonSchema()` exports. Fixed `$NODE_ID` → `$SPECIALISTS_NODE_ID` in all renderers.

### Env var injection (src/pi/session.ts + src/specialist/runner.ts)
Added `env` field to `PiSessionOptions`. Runner forwards `SPECIALISTS_NODE_ID` and `SPECIALISTS_JOB_ID` from specialist variables to real shell env vars in the pi process. This enables coordinator bash calls to resolve `$SPECIALISTS_NODE_ID`.

### Coordinator config (config/specialists/node-coordinator.specialist.json)
Permission READ_ONLY → LOW. Removed response_format, output_type, output_schema. System prompt rewritten with CLI command reference, execution loop, and few-shot examples. Required tools now include `bash`. Version 1.2.0 → 1.3.0.
