## Code Changes

### Reviewer startup contract
- `src/cli/run.ts` now injects real `reviewer_diff_source`, `reviewer_diff_stat`, `reviewer_diff_files`, and `reviewer_diff_hunks` for reviewer `--job` runs. This makes injected diff context real input data, not prompt fiction.
- Same CLI helper now truncates per-file and total hunk payload size, preventing `E2BIG` when reviewer starts on large patches.
- `src/specialist/runner.ts` now parses injected reviewer diff payload, prioritizes it ahead of git fallbacks, and preserves fallback order: injected → unstaged → staged → branch-vs-base. Error text now matches actual source chain.

### Tests
- `tests/unit/specialist/runner.test.ts` now covers three startup invariants: staged fallback works, all-empty sources hard-fail, injected diff context wins over git fallback.
- Old prompt-equality tests were updated to tolerate `MANDATORY_RULES` append behavior, which was causing unrelated failures and muddying review signal.

### Runtime artifacts
- `dist/index.js`, `dist/cli/run.js`, `dist/specialist/runner.js`, related `.d.ts` / sourcemap outputs updated from rebuild. Final merge contained only relevant generated artifacts after unrelated dist churn was restored away before merge.
