# Session Report — 2026-03-31

## Summary

A full-stack delivery session: planned, implemented, reviewed, and shipped the tmux-backed background session feature (unitAI-0g8g → unitAI-uh5s), three-wave specialist orchestration with reviewer gates at every wave (all PASS, scores 88–98), published 3.4.1 through 3.4.4, and closed eight additional issues across MCP cleanup, docs sync, and two critical bugs. The tmux implementation gives `--background` runs named sessions (`sp-<specialist>-<jobid>`), accessible via `specialists attach` or `Ctrl-b s`; graceful fallback when tmux is absent. The MCP surface was reduced from 11 tools to 1 (`use_specialist` only) — `feed_specialist` was hanging Claude, `start_specialist` used in-memory JobRegistry invisible to CLI, and the general pattern of Claude orchestrating via MCP was unreliable. A critical init guard (`shouldSkipDefaultSyncInPiSession()`) was added to `src/cli/init.ts` after executors running inside pi sessions repeatedly wiped `.specialists/default/` by invoking `specialists init`. Two new issues filed for next session: keep-alive hardening (sync-docs went to `done` instead of `waiting`) and tmux window nesting (new sessions should appear as windows inside the current session, not top-level).

## Code Changes

### New commands: attach + list --live (tmux feature)
`src/cli/attach.ts` — reads `tmux_session` from `status.json`, calls `tmux attach -t <session>`. `src/cli/tmux-utils.ts` — `isTmuxAvailable()`, `buildSessionName()` (canonical `sp-<specialist>-<suffix>`), `createTmuxSession()` (injects `SPECIALISTS_TMUX_SESSION` env), `killTmuxSession()` (idempotent). `src/cli/list.ts` — `--live` flag: `@inquirer/select` arrow-key selector with enter-to-attach; plain text fallback for non-TTY.

### Background run wiring (src/cli/run.ts + supervisor.ts)
`--background` calls `createTmuxSession()` when tmux available; falls back to detached spawn silently. Supervisor reads `SPECIALISTS_TMUX_SESSION` into `status.json.tmux_session?`; calls `killTmuxSession()` in finally block. `writeReadyMarker()` refactored into private method — now fires on both success and error paths.

### MCP reduction (src/server.ts)
`this.tools = [createUseSpecialistTool(runner)]`. 10 tools removed from registration. `start_specialist.tool.ts` got `[DEPRECATED]` in description and return value with warning pointing to `--background`.

### Init guard (src/cli/init.ts)
`shouldSkipDefaultSyncInPiSession()` — checks `SPECIALISTS_TMUX_SESSION`, `PI_SESSION`, `SPECIALISTS_INIT_FORCE_DEFAULT_SYNC` env vars, plus process ancestry walk for `pi` binary. When true + existing defaults present, skips `copyCanonicalSpecialists()`. PostToolUse hook for `specialists-complete.mjs` now wired by init.
