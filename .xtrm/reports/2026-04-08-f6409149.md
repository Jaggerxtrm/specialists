# Session Report — 2026-04-08

## Summary

Validated the full observability pipeline against the pi RPC event surface. The initial audit (explorer) mapped 8 DB tables and 17 timeline event types against 28 RPC commands and 14 RPC events, identifying 6 gaps. During implementation, three additional bugs surfaced: (1) `initSchema()` unconditionally DROP/RENAME-ing `specialist_jobs` on every init, creating a race window visible as "no such table" errors in `sp feed`; (2) `findTokenUsage()` in `session.ts` only parsing OpenAI-style token fields while pi RPC emits `input/output/cacheRead/cacheWrite` with nested `cost.total` — the root cause of token_usage always showing 0%; (3) specialist loader resolving from `.specialists/default` instead of `config/specialists`, causing `sp edit` model changes to be ignored. All three bugs were fixed, new timeline events added (extension_error, model_change, enriched compaction/retry), token display added to `sp ps` and `sp result`, and docs synced.
