## Problems Encountered

| Problem | Root Cause | Resolution |
|---------|-----------|------------|
| Executor dies during `bun vitest run` | PiAgentSession stall detection triggers SIGKILL during tinypool worker initialization (silent startup phase). 5-level subprocess chain: codex → pi → bash → bun → vitest → tinypool. | Implemented test-aware stall detection (we7w): regex-based test command detection extends stall window to 300s. |
| SQLite "Cannot use a closed database" spam | Async ops (setInterval, FIFO callbacks, Promise microtasks) fire after Supervisor.dispose() closes the SQLite DB. 5-attempt retry loop masks it under bun (slower teardown) but surfaces under node (faster teardown). | Pending-ops tracker + async dispose() (fjf0). Eliminated all error instances. bun vitest 2.3× faster. |
| Reviewer specialist stuck in waiting indefinitely | run_complete event blocked behind keepAliveExitPromise (only emitted at session teardown, not per-turn). No self-termination for READ_ONLY specialists after terminal verdict. | Per-turn run_complete emission + TERMINAL_COMPLIANCE_VERDICT_REGEX auto-close for READ_ONLY + reviewed_job_id plumbing (wagb/n5e9). |
| sync-docs writes to main repo instead of worktree | Pi edit tool honors absolute paths regardless of specialist cwd. LLMs commonly generate absolute paths from training. | Write-side boundary enforcement via generated pi extension (a2u7/elhl). Validated: subsequent sync-docs stayed inside worktree. |
| Debugger specialist hangs on first turn (0 tools, 0 tokens) | Same keep-alive first-turn race as reviewer hang. Model call never fires before waiting flag is set. | Fixed by wagb supervisor lifecycle changes. |
| console.error() not captured by process.stderr.write spy in vitest tests | Node.js console.error has its own write path that doesn't route through spied process.stderr.write. | Use vi.spyOn(console, 'error') for tests that assert on console.error output. Memory saved. |
