# Session Report — 2026-04-26

## Summary

Designed, implemented, smoke-tested, packaged, and documented `specialists-service` v1 — a thin HTTP wrapper around `pi` exposing script-class specialists over `POST /v1/generate`. The session moved through four phases: (1) design critique — explorer + two adversarial overthinker passes produced a 713-line evaluation memo (`docs/specialists-service-evaluation.md`) that surfaced 11 production blockers and forced explicit rejection of the original draft's schema-fork approach; (2) reset — user pushback on over-engineering compared the proposal to qwen-service's tiny wrapper, leading to a focused 172-line spec (`docs/specialists-service.md`) that delegates credentials to pi via `~/.pi` bind-mount and reuses the existing `SpecialistLoader` / `observability-sqlite` writers without schema changes; (3) implementation — executor chain `97cfc3 → 67ee60 → c9fa4b` with one FAIL-then-PASS review loop landed `sp serve`, a one-shot script-runner that bypasses `SpecialistRunner`, real semaphore queue, SIGTERM drain forwarding to child pi subprocesses, 4MB stdout cap, and an integration test that asserts both response shape and DB row presence (495 LOC, 6 tests passing); (4) packaging + docs — bun-runtime Docker image with non-root UID 10001 and pinned pi 0.64.0, sidecar compose example, build-from-repo install guide with rootless-podman + Fedora-SELinux notes drawn from real testing, schema reference section in `docs/authoring.md`, and a Schema target translation table in `docs/darth-feedor-migration.md` showing how legacy YAMLs convert to the new JSON shape. Three real bugs surfaced via smoke testing and were fixed inline — pi NDJSON shape was misparsed (test mock perpetuated fiction), pi-level `errorMessage` was lost when content was empty (silent failure on quota exhaustion), and JSON-mode responses with markdown fence wrapping (`​```json … ​````) failed to parse. Two durable findings saved to memory: the Zod schema strips unknown keys on parse (verified bug, pre-existing, breaks `sp edit` round-trip on any spec with extra fields), and four packaging gotchas around the image (pi version pin, bun-vs-node runtime incompatibility, UID collision with `node:lts-slim`, rootless podman SELinux). Eight follow-up beads filed under a new deferred-items epic so the v1 minimum stays clean and future work has explicit task contracts.

## Problems Encountered

| Problem | Root Cause | Resolution |
|---|---|---|
| Pi NDJSON parser returned empty text → `JSON Parse error: Unexpected EOF` | `extractAssistantText` only matched a fictional `{type:"assistant",data:{text}}` shape. Real pi emits `{type:"message_end",message:{role:"assistant",content:[{type:"text",text}]}}` and `{type:"agent_end",messages:[...]}`. Test mock matched the parser, not reality, so the regression was invisible. | Rewrote parser to handle both real shapes; updated test mock; added pi-level `errorMessage` extraction so quota/auth errors surface correctly when content is empty. |
| Models wrap JSON in markdown fences (kimi did `​```json …​````) → JSON parse fails on backtick | LLMs default to fenced output; `response_format=json` doesn't actually force raw JSON. | Added `stripMarkdownFences()` before `JSON.parse` for `response_format=json` responses. |
| Pi exit 0 with empty content but populated `errorMessage` → returned `success:true` with empty output | Anthropic quota exhausted produced this exact pattern. The runner trusted exit code instead of inspecting the final message. | Added `extractPiErrorMessage` that walks NDJSON for `message.errorMessage`; mapped through `mapErrorType()` (added "out of extra usage", "insufficient_quota", "401", "429" patterns). |
| Container exited immediately with `TypeError: __require is not a function` | Bun build emits a `__require` shim incompatible with plain node. Runtime stage was `node:lts-slim` running `node /app/dist/index.js`. | Switched runtime base to `oven/bun:1.3.5-slim`; sp shim now `exec bun /app/dist/index.js`. |
| `useradd: UID 1000 is not unique` during image build | `node:lts-slim` already ships a UID 1000 user. | Bumped image's non-root user to UID 10001; updated `org.specialists.uid` label and install-guide pitfalls section. |
| Container hit `EACCES: permission denied, mkdir '/work/.specialists/db'` despite host dir being `1000:1000 755` | Running with default rootless podman maps container UID 1000 to a host subuid, which can't write into the host's 1000-owned dir. | Documented `--userns=keep-id --user "$UID:$GID"` combo (Fedora-specific) in install guide. Standard Docker only needs `--user "$UID:$GID"`. |
| Same dir was inaccessible even after fixing UID — `stat` showed `1000:1000 755` but `ls` got Permission denied | Fedora SELinux blocks bind mounts without relabeling. | Documented `:z` SELinux relabel suffix in install guide. |

| WORKDIR `/app` was read-only for UID 10001 → service tried to create `/app/.specialists/` | Build needs `/app`; runtime needs `/work` (where bind mounts land). | Set explicit `WORKDIR /work` in the runtime stage after USER directive. |

## Code Changes

### sp serve runtime (3 files, 357 lines)

- **`src/cli/serve.ts`** (new, 120 lines) — Node `http` server on `--port`. Real semaphore via `waitForSlot` polling at 25ms up to `--queue-timeout-ms` (default 5s) before returning HTTP 429. SIGTERM handler maintains a `Set<ChildProcess>` populated via `onChild` callback, forwards SIGTERM to all in-flight pi children, drains up to `--shutdown-grace-ms`, then SIGKILL stragglers. Uses canonical observability path resolver (`resolveObservabilityDbLocation`) so the trace DB lives at `<git-root>/.specialists/db/observability.db` — same file `sp run` writes to. `/healthz` returns process-alive only.
- **`src/specialist/script-runner.ts`** (new, 211 lines) — `compatGuard` rejects specs that fail the script-class boundary (`interactive`, `requires_worktree`, non-READ_ONLY, `skills.scripts` non-empty). `renderTaskTemplate` raises `template_variable_missing` on any un-substituted `$ident`. Pi spawn uses arg array with `--mode json --no-session --no-extensions --no-tools --model X [--thinking L] -- "<prompt>"`. Stdout capped at 4MB → `output_too_large` (kills pi via SIGTERM then SIGKILL after 2s grace). NDJSON parser handles `message_end` and `agent_end` shapes. Markdown fence stripping for `response_format=json`. Trace row writes via the existing `observability-sqlite` client with `surface: 'script_specialist'` inside `status_json` so script calls are queryable separately from `sp run` rows. Closed error taxonomy union (`specialist_not_found | specialist_load_error | template_variable_missing | auth | quota | timeout | network | invalid_json | output_too_large | internal`).
- **`src/index.ts`** (+19 lines) — `serve` subcommand dispatch.

### Tests (2 files, 138 lines)

- **`tests/unit/specialist/script-runner.test.ts`** (new, 37 lines) — 4 compat-guard rejection cases (interactive, requires_worktree, non-READ_ONLY, scripts non-empty) + 1 missing-variable assertion.
- **`tests/integration/sp-serve.test.ts`** (new, 108 lines) — Spawns `bun src/index.ts serve` as a real subprocess on a random port, builds a temp `.specialists/user/echo.specialist.json` and a fake `pi` binary that emits a real-shape NDJSON event, sends `POST /v1/generate` via `fetch`, asserts response shape AND opens the resulting SQLite DB to assert one row with `specialist='echo'` and `status_json.surface='script_specialist'`.

### Image + deployment (3 files, 60 lines)

- **`Dockerfile`** (new, 33 lines) — Multi-stage: `oven/bun:1.3.5` builder (frozen lockfile + `bun run build`) → `oven/bun:1.3.5-slim` runtime that installs pinned `@mariozechner/pi-coding-agent@0.64.0` via npm, creates UID 10001 `specialists` user, copies `dist/`, writes `sp` shell shim that execs `bun /app/dist/index.js`. `WORKDIR /work` so observability resolver lands at `/work/.specialists/db/observability.db`. `LABEL org.specialists.uid="10001"`. `ENTRYPOINT ["sp", "serve"]`, `CMD ["--port", "8000"]`.
- **`.dockerignore`** (new, 10 lines) — Excludes `node_modules`, `.git`, `.specialists/jobs`, `.specialists/db`, `.worktrees`, `.beads`, `dist`, `tests`, `docs`, `*.md`.
- **`docker/compose.example.yml`** (new, 29 lines) — Sidecar template. `image: specialists-service:local` with commented `build:` block for build-from-repo. `user: "${UID:-1000}:${GID:-1000}"`. Bind-mounts `./.specialists:/work/.specialists` and `${HOME}/.pi:/pi-home/.pi:ro`. `HOME=/pi-home` so pi resolves auth from the mounted home. Inline notes for SQLite + bind mount and Fedora-podman variants.
