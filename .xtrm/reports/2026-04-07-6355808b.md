## Code Changes

### Installer Architecture (`src/cli/init.ts`)
Major rewrite of skill and hook installation. Skills now flow through `.xtrm/skills/default/` with active symlinks instead of direct copy. Hooks install to `.xtrm/hooks/specialists/` with `.claude/hooks/` symlinks. Added `assertXtrmPrerequisites()` (checks `.xtrm/` dir + `xt` CLI), `assertSkillRootSymlink()` (validates `.claude/skills` and `.pi/skills` are proper symlinks), `ensureActiveSkillSymlink()` (creates per-skill symlinks in active dirs). New `InitOptions`: `syncSkills`, `noXtrmCheck`.

### Process Cleanup (`src/cli/clean.ts`, `src/specialist/supervisor.ts`)
Extended `clean` with `--processes` flag to kill all running/starting specialist jobs. Supervisor now writes `binary_version` to `status.json` on job start. SessionStart hook (`config/hooks/specialists-session-start.mjs`) warns about stale binary versions with actionable `clean --processes` suggestion.

### Drift Detection (`src/cli/doctor.ts`)
Added skill drift checks: hash comparison between `config/skills/` and `.xtrm/skills/default/`, symlink chain validation, broken/missing link detection. Reports drifted skills with fix commands.

### CLI (`src/index.ts`)
Added `--sync-skills` and `--no-xtrm-check` flags to init command. Updated help text with symlink prerequisite documentation.

### Surface Ownership Doc (`docs/surface-ownership.md`)
New file documenting every installed artifact's canonical owner (specialists init vs xtrm install), the shared file protocol for `.claude/settings.json`, and the source-of-truth chain.

### Test Fixes
`tests/unit/cli/init.test.ts`: Added `setupXtrmStructure()` helper that creates full `.xtrm` directory + symlinks before each test. All tests pass `noXtrmCheck: true`.
`tests/integration/cli/init.integration.test.ts`: Same structure setup + `SPECIALISTS_INIT_FORCE=1` for non-TTY.
