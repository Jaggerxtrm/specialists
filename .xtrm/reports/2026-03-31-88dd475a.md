## Code Changes

### Feed follow-mode fix (`src/cli/feed.ts`)
`followMerged()` now builds a `trackedJobs` set of only non-terminal jobs at follow-start. Exit condition checks `trackedJobs.size > 0 && completedJobs.size === trackedJobs.size` rather than comparing against total job count — prevents early exit when already-done jobs exist. `makeJobMetaReader(jobsDir, { useCache: false })` added so each poll cycle reads fresh `status.json` via `openSync/closeSync`, enriching events with current model/backend. `readFileFresh()` helper uses `openSync`/`closeSync` to bypass OS file descriptor cache in poll mode.

### Test isolation fixes (`tests/unit/cli/attach.test.ts`, `result.test.ts`, `feed.test.ts`)
All three tests had `const specialistsDir = join(process.cwd(), '.specialists')` at module scope with `rmSync` in `beforeEach`/`afterEach` pointing at the real project directory. Fixed by switching to `mkdtempSync(join(tmpdir(), 'sp-*-test-'))` per-test temp dirs with `vi.spyOn(process, 'cwd').mockReturnValue(tempRoot)`. feed.test.ts was fixed in the prior session; attach + result fixed this session after debugger identified them.
